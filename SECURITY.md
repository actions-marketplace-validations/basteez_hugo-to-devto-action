# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest `v1.x.y` | ✅ |
| Older `v1.x.y` | ❌ (upgrade to the latest `v1.x.y`) |

## Reporting a Vulnerability

**Primary channel**: Use GitHub's private vulnerability reporting on this repository:
<https://github.com/basteez/hugo-to-devto-action/security/advisories/new>

**Fallback channel**: If private reporting on GitHub is not available to you, email: `tiz.basile+security@gmail.com`

Please do NOT file public GitHub issues for suspected vulnerabilities.

## What to expect

- Acknowledgement within 7 days.
- Best-effort triage and fix. No SLA.
- Credit in the resulting GitHub security advisory (if desired).
